# Protocol Durable Record Schemas (v1)

This document defines the canonical runtime-validated payload shapes for all durable protocol record types.

## Shared rules

- Every durable record includes:
  - `record_type`: string discriminator literal for the specific record family.
  - `record_version`: fixed numeric literal `1`.
- Signatures are explicit fields in each payload.
- All records are append-only evidence payloads and do not represent mutable current state.

## `identity_binding`

**Purpose**
- Asserts that a subject UUID is bound to a subject identity public key under key possession proof.

**Canonical field list**
- `record_type`
- `record_version`
- `subject_uuid`
- `subject_identity_public_key`
- `key_epoch`
- `created_at`
- `self_signature`

**Field definitions**
- `record_type`: literal string `"identity_binding"`.
- `record_version`: literal number `1`.
- `subject_uuid`: UUID string for the subject.
- `subject_identity_public_key`: public key string.
- `key_epoch`: non-negative integer key epoch counter.
- `created_at`: timestamp string.
- `self_signature`: signature string over the record payload.

**Signature semantics**
- `self_signature` is produced by the subject key corresponding to `subject_identity_public_key`.

**Example**
```json
{
  "record_type": "identity_binding",
  "record_version": 1,
  "subject_uuid": "6f1a6eaf-f6d6-4d8c-a5e0-3ddf2b4531a7",
  "subject_identity_public_key": "did:key:z6Mki4R3K1x",
  "key_epoch": 1,
  "created_at": "2026-03-15T10:00:00.000Z",
  "self_signature": "sig_identity_binding_self"
}
```

## `endorsement`

**Purpose**
- Expresses a signed binary statement from one binding about another binding (`binding_valid` or `binding_invalid`).

**Canonical field list**
- `record_type`
- `record_version`
- `endorser_binding_hash`
- `subject_binding_hash`
- `endorsement_type`
- `signature`

**Field definitions**
- `record_type`: literal string `"endorsement"`.
- `record_version`: literal number `1`.
- `endorser_binding_hash`: hash string for endorser identity binding.
- `subject_binding_hash`: hash string for subject identity binding.
- `endorsement_type`: enum string, one of:
  - `"binding_valid"`
  - `"binding_invalid"`
- `signature`: signature string by the endorser.

**Signature semantics**
- `signature` is generated by the endorser corresponding to `endorser_binding_hash`.

**Example**
```json
{
  "record_type": "endorsement",
  "record_version": 1,
  "endorser_binding_hash": "hash_endorser_binding",
  "subject_binding_hash": "hash_subject_binding",
  "endorsement_type": "binding_valid",
  "signature": "sig_endorsement"
}
```

## `handshake`

**Purpose**
- Captures mutually signed evidence that two participants completed a proximity interaction.

**Canonical field list**
- `record_type`
- `record_version`
- `handshake_uuid`
- `participant_a_binding_hash`
- `participant_b_binding_hash`
- `participant_a_merkle_root`
- `participant_b_merkle_root`
- `ephemeral_keys`
- `signatures`

**Field definitions**
- `record_type`: literal string `"handshake"`.
- `record_version`: literal number `1`.
- `handshake_uuid`: UUID string.
- `participant_a_binding_hash`: hash string.
- `participant_b_binding_hash`: hash string.
- `participant_a_merkle_root`: hash string.
- `participant_b_merkle_root`: hash string.
- `ephemeral_keys`: object with string keys:
  - `participant_a`
  - `participant_b`
- `signatures`: object with string signatures:
  - `participant_a`
  - `participant_b`

**Signature semantics**
- `signatures.participant_a` and `signatures.participant_b` are each participant's signature over the same handshake payload.

**Example**
```json
{
  "record_type": "handshake",
  "record_version": 1,
  "handshake_uuid": "680a3e96-1f84-4c8b-8b39-b664b1744d43",
  "participant_a_binding_hash": "hash_binding_a",
  "participant_b_binding_hash": "hash_binding_b",
  "participant_a_merkle_root": "hash_merkle_root_a",
  "participant_b_merkle_root": "hash_merkle_root_b",
  "ephemeral_keys": {
    "participant_a": "ephemeral_pubkey_a",
    "participant_b": "ephemeral_pubkey_b"
  },
  "signatures": {
    "participant_a": "sig_handshake_a",
    "participant_b": "sig_handshake_b"
  }
}
```

## `key_rotation`

**Purpose**
- Links a previous binding to a new binding for the same subject while preserving history.

**Canonical field list**
- `record_type`
- `record_version`
- `subject_uuid`
- `old_binding_hash`
- `new_binding_hash`
- `rotation_counter`
- `signatures`

**Field definitions**
- `record_type`: literal string `"key_rotation"`.
- `record_version`: literal number `1`.
- `subject_uuid`: UUID string.
- `old_binding_hash`: hash string.
- `new_binding_hash`: hash string.
- `rotation_counter`: non-negative integer.
- `signatures`: object with string signatures:
  - `old_key`
  - `new_key`

**Signature semantics**
- `signatures.old_key` is made by the key represented by `old_binding_hash`.
- `signatures.new_key` is made by the key represented by `new_binding_hash`.

**Example**
```json
{
  "record_type": "key_rotation",
  "record_version": 1,
  "subject_uuid": "6f1a6eaf-f6d6-4d8c-a5e0-3ddf2b4531a7",
  "old_binding_hash": "hash_old_binding",
  "new_binding_hash": "hash_new_binding",
  "rotation_counter": 2,
  "signatures": {
    "old_key": "sig_rotation_old_key",
    "new_key": "sig_rotation_new_key"
  }
}
```

## `revocation`

**Purpose**
- Marks a previously emitted record as revoked by a signer.

**Canonical field list**
- `record_type`
- `record_version`
- `signer_binding_hash`
- `target_record_hash`
- `reason_code`
- `signature`

**Field definitions**
- `record_type`: literal string `"revocation"`.
- `record_version`: literal number `1`.
- `signer_binding_hash`: hash string for the revoking signer binding.
- `target_record_hash`: hash string of the revoked target record.
- `reason_code`: enum string, one of:
  - `"key_compromised"`
  - `"superseded"`
  - `"consent_withdrawn"`
  - `"other"`
- `signature`: signature string by the revoking signer.

**Signature semantics**
- `signature` is made by the signer identified by `signer_binding_hash`.

**Example**
```json
{
  "record_type": "revocation",
  "record_version": 1,
  "signer_binding_hash": "hash_signer_binding",
  "target_record_hash": "hash_target_record",
  "reason_code": "key_compromised",
  "signature": "sig_revocation"
}
```
